# 游戏交互设计：让玩家真正想动手

这一课不打算背“交互设计原则”，而是直接讨论几个更实际的问题：

- 什么才算交互？
- 为什么有些东西一看就想碰，有些东西按完毫无感觉？
- 《超阈限空间》和《看门狗》的核心交互到底有趣在哪里？
- 经典游戏中的交互是怎样实现的？
- 如果自己做游戏，还能设计哪些不一样的交互？

---

## 什么是交互

先看一个最普通的场景：

玩家走到门前，屏幕显示“按 E 开门”。玩家按下 E，门打开。

这当然是交互，但它只完成了最基本的一件事：玩家发出命令，游戏执行命令。它能用，却不一定有趣。

再换一个场景：

玩家发现门太小，自己过不去。旁边有一个模型房子，拿起来以后放到远处，模型房子突然变成真正大小；玩家再把门拿近，门又缩成手掌那么大。此时玩家不是在“执行开门命令”，而是在试探世界的规则。

这就是更有意思的交互。

一句话解释：

> 交互就是玩家动手做一件事，游戏世界给出回应，并且这个回应会影响玩家接下来想做什么。

它至少有四步：

```text
玩家产生想法
→ 做出操作
→ 世界发生变化
→ 玩家看到结果，产生下一个想法
```

例如玩家把箱子推到墙边：

- 他想爬上高处；
- 他推动箱子；
- 箱子真的移动并停在墙边；
- 他踩上箱子，发现能够到达原本够不到的平台。

这里真正重要的不是“按了哪个键”，而是玩家发现：箱子不仅是装饰，它可以改变路线。

### 输入不等于交互

按键、鼠标、手柄都只是输入。输入被游戏解释之后，才产生交互。

同样是按鼠标左键，可以是：

- 开枪；
- 抓住物体；
- 在墙上画一条路；
- 选择监控摄像机；
- 把时间停在某一帧；
- 把两个物体的材质互换。

所以设计交互时，不要先问“这个功能放在哪个键”，先问：

```text
玩家到底在改变什么？
```

### 好交互像一场对话

玩家用动作向游戏提问：

- 这个杯子能拿起来吗？
- 这个火能不能点燃木头？
- 我从这个摄像机里能不能控制那扇门？
- 我把物体放得更远，它会不会变大？

游戏通过结果回答：

- 能，杯子被拿起来了；
- 能，木头开始冒烟并燃烧；
- 能，门的控制图标亮了；
- 会，而且它现在大到可以当平台。

如果游戏的回答清楚，玩家就愿意继续问。如果游戏经常不回答、回答得莫名其妙，玩家很快就不想碰了。

## 怎样让交互吸引人

### 让玩家一眼就想试

会动的东西、发光的按钮、奇怪的声音、与周围不同的材质，都能吸引注意。

但不能把所有东西都做得很亮。满屋子都是高亮，相当于什么都没有高亮。最好只突出当前真正值得玩家尝试的对象。

例如：

- 普通墙面保持安静，能推动的墙砖轻微凸出；
- 监控摄像机缓慢转动，镜头偶尔闪过红光；
- 一个看似普通的箱子，在玩家靠近时影子却不符合它的大小；
- 门后传来机械声，暗示它并不是普通的装饰门。

玩家还没看到文字提示，就已经想过去看看，这比满屏“按 E”更有效。

### 玩家动手以后，世界要立刻回应

一扇沉重的门可以慢慢打开，但按键的一瞬间必须有反应：

- 门把手先压下；
- 锁舌发出声音；
- 门轻微震动；
- 角色的手碰到门；
- 然后门再缓慢转动。

如果玩家按下以后半秒钟什么都没有发生，他通常会再按一次。接着门可能连续触发两次，看起来就像程序坏了。

完整动作可以慢，确认反馈不能含糊。

### 不要只给一种用途

一个物体只能在指定位置按一下，玩家会把它当成钥匙。

一个物体可以：

- 堵门；
- 垫脚；
- 压住机关；
- 挡住激光；
- 从高处扔下砸碎玻璃；

玩家才会把它当成真正存在于世界中的东西。

好玩的交互经常不是“功能很多”，而是一个简单功能能在不同场景里产生不同结果。

### 规则要稳定，结果可以出人意料

惊喜不等于随机。

《传送门》中“穿过入口会从出口出来”始终成立。真正的惊喜来自玩家后来发现，速度也会被保留，于是可以从高处落下，再从另一面高速飞出去。

规则没有突然改变，只是玩家发现了它更深的一层用途。

如果同一个红色按钮有时开门、有时关灯、有时让玩家死亡，而且没有任何提示，这不是惊喜，只是规则混乱。

### 给玩家留一点“乱玩”的空间

玩家经常不会按照设计者预想的方式行动。他们会：

- 把任务物品扔到奇怪的位置；
- 用箱子堵住 NPC；
- 连续快速开关机关；
- 试图从关卡边缘爬出去；
- 把一个看似只用于解谜的东西拿去战斗。

如果这些尝试能产生合理结果，玩家会觉得这个世界“真的能玩”。不必保证每种乱玩都有奖励，但至少不要让系统立刻崩坏或毫无反应。

### 让交互改变局面

按钮亮一下、音效响一下，却没有任何实际影响，这种交互很快会失去吸引力。

一次好的交互应该至少改变一件事：

- 路线改变；
- 物体状态改变；
- 敌人行为改变；
- 玩家获得新的观察角度；
- 原本不可能的动作变得可能；
- 玩家对世界规则的理解改变。

玩家愿意继续操作，是因为他相信自己的动作能够留下结果。

### 逐步把同一规则玩出新花样

一个新交互可以这样安排：

```text
第一次：只需要发现它能用
第二次：用它解决一个简单问题
第三次：和另一个机制组合
第四次：在压力下使用
第五次：反过来利用它
```

例如“物体放远会变大”：

- 第一次，把小方块放远，变成台阶；
- 第二次，把大障碍拿近，缩小后移走；
- 第三次，用放大的物体挡住激光；
- 第四次，在地面不断下沉时快速寻找落点；
- 第五次，故意把门缩小，让追来的敌人无法通过。

这时同一个交互不需要增加新按键，也能不断产生新鲜感。

## 《超阈限空间》：把透视错觉变成工具

### 它的主要交互是什么

游戏里最有代表性的交互，是拿起一个物体，然后利用透视关系改变它的真实大小。

玩家抓住物体时，物体在屏幕里看起来保持差不多大。把它放到远处，因为远处的东西本来应该显得更小，游戏就会把物体的真实尺寸放大；把它放到眼前，物体的真实尺寸就会缩小。

于是一个巴掌大的棋子可以被放成楼梯，一块巨大的木板也可以缩小后从门里带走。

### 有趣在哪里

它利用的是玩家非常熟悉的视觉经验：近大远小。

平时我们知道远处的汽车并没有真的变小，只是看起来小。《超阈限空间》反过来利用这件事：既然它在屏幕上看起来一样大，那就让它在世界里真的变大。

有趣的地方不只是“物体会缩放”，而是玩家必须用摄像机寻找合适的距离和角度。视角不再只是观察工具，它直接参与解谜。

这个机制还能自然产生多种用途：

- 把小东西变成平台；
- 把大东西缩小后搬走；
- 改变物体与门洞的尺寸关系；
- 用变大的物体压住机关；
- 从特定角度制造一个看似不可能的落点。

### 简单实现思路

先限定只有标记为可抓取的简单物体能够使用这个功能。

抓取时，从摄像机中心发射射线，命中物体后记录：

- 初始缩放 `startScale`；
- 物体与摄像机之间的初始深度 `startDepth`；
- 抓取前的位置，用于失败时复位。

手持期间，继续沿摄像机中心射线寻找放置位置。新的缩放可以先用一个很简单的关系计算：

```csharp
float ratio = currentDepth / startDepth;
Vector3 newScale = startScale * ratio;
```

物体放得越远，`currentDepth` 越大，缩放也越大；放得越近，缩放越小。

为了让结果稳定，还要做几件事：

- 抓取期间让 Rigidbody 进入 Kinematic，避免物理系统把物体甩走；
- 使用等比缩放，不要分别改变 X、Y、Z；
- 根据表面法线把物体向外推一点，避免嵌入墙面；
- 使用 `Physics.OverlapBox` 或 `Physics.ComputePenetration` 检查整个物体是否放得下；
- 合法位置显示正常轮廓，非法位置显示红色幽灵模型；
- 限制最小和最大尺寸；
- 没有合法位置时，不允许松手，或把物体送回上一个安全位置。

最初原型只需要一个 Cube、一台透视摄像机、一面墙和一个高台。先确认“放远变大、拿近变小”真的好玩，再增加旋转、复杂模型和更多关卡规则。

## 《看门狗》：让控制权沿着摄像机网络移动

### 它的主要交互是什么

玩家可以接入监控摄像机，从摄像机画面中寻找另一台摄像机，再继续切换过去。虽然角色的身体还在原地，玩家的观察范围和控制范围却沿着网络进入了远处空间。

从监控画面里，玩家还能操作环境设备，例如：

- 打开或关闭门；
- 控制升降机；
- 切换灯光；
- 触发机关；
- 制造声音吸引敌人；
- 打开新的摄像机视角。

玩家不是亲自走进危险区域，而是先通过摄像机观察路线，再远程改变环境。

### 有趣在哪里

它把“看见”变成了一种能力。

普通摄像机只是让玩家看画面，而这里的摄像机是通往其他设备的入口。只要找到正确角度，就可能隔着墙、楼层甚至敌人控制远处的东西。

这种交互会让一个普通场景变成网络迷宫：

```text
我先找到摄像机 A
→ 从 A 看见摄像机 B
→ 从 B 看见门的控制器
→ 打开门
→ 玩家本体再通过这条路线
```

它有趣的另一个原因是，玩家身体和视线暂时分开了。玩家在远处操作时，自己的角色可能仍然处于危险中，这会带来紧张感。

### 简单实现思路

每台摄像机做成一个 `CameraNode`，记录：

- 自己的 Camera；
- 是否在线；
- 是否已经解锁；
- 能切换到哪些其他节点；
- 当前画面里可以控制哪些设备。

场景中只让当前摄像机真正渲染。切换时关闭旧 Camera，开启新 Camera，并把输入从“控制角色”切换成“旋转摄像机、选择目标、退出监控”。

摄像机之间可以用两种简单规则连接：

- 固定连接：在 Inspector 中直接配置下一台摄像机；
- 视线连接：从当前摄像机发射射线，只有真正看见目标摄像机时才能切换。

门、灯和升降机不要直接写在摄像机脚本里，可以统一实现一个远程控制接口：

```csharp
public interface IRemoteDevice
{
    bool CanUse();
    string GetActionName();
    void Use();
}
```

摄像机只负责寻找目标并调用 `Use()`。门自己处理开关动画，灯自己切换亮灭，升降机自己判断是否正在运动。

实现时最容易漏掉三件事：

- 任何时候都要允许玩家退出监控画面；
- 门在动画中不能被重复触发到状态混乱；
- 远程画面里要显示当前摄像机名称、可控目标和玩家本体是否受到威胁。

## 《超级马力欧》：跳跃为什么一直好玩

### 交互细节

马力欧的跳跃不是角色突然获得一个固定向上的速度那么简单。

玩家按键的时间会影响高度：轻点是短跳，按住是高跳。离开平台边缘的一小段时间内，游戏通常仍然允许起跳；稍微提前按下跳跃，角色落地时也可以立刻跳起来。这些隐藏的宽容让玩家觉得自己按得很准。

跳跃还会同时作用于很多东西：

- 越过坑洞；
- 踩敌人；
- 顶砖块；
- 收集空中的金币；
- 改变移动节奏；
- 选择安全路线或危险捷径。

一个动作同时服务移动、战斗和收集，所以它不会只用一次就失去意义。

### 实现思路

基础实现需要保存竖直速度：

- 按下跳跃时给一个向上的初速度；
- 每帧累加重力；
- 松开跳跃键时，如果角色仍在上升，就削弱竖直速度，形成短跳；
- 落地时把竖直速度归零。

手感优化可以加入：

- Coyote Time：角色离开平台后的短时间内仍可起跳；
- Input Buffer：落地前稍早按下的跳跃会被短暂保存；
- 上升和下落使用不同重力，让下落更干脆；
- 接近跳跃顶点时稍微降低重力，让玩家更容易调整落点；
- 空中保留一定水平控制，但不要和地面完全一样。

踩敌人时检测玩家是否从上方向下接触敌人。如果成立，敌人受击，玩家获得一个较小的反弹速度；从侧面接触则由玩家受伤。

## 《传送门》：门没有移动，空间关系变了

### 交互细节

玩家可以在特定表面放置两个传送门。进入一个门，会从另一个门出来。

真正让这个机制变得精彩的，不是“从 A 传送到 B”，而是方向和速度也会跟着转换。玩家可以从高处落下获得速度，再从墙上的传送门水平飞出去。

这让“掉下去”从失败变成一种加速方式，也让地面、墙面和天花板重新获得新的连接关系。

### 实现思路

两个 Portal 互相保存引用。物体穿过入口平面时：

1. 计算物体相对入口 Portal 的局部位置和旋转；
2. 把这个局部变换转换到出口 Portal；
3. 将物体移动到出口对应位置；
4. 把 Rigidbody 的速度也从入口坐标系转换到出口坐标系；
5. 暂时记录刚传送过的物体，避免它在同一帧又被传回去。

门内画面可以让出口摄像机把画面渲染到 Render Texture，再显示在入口表面。更完整的效果需要根据主摄像机与入口的相对位置实时摆放出口摄像机，并处理递归传送门、近裁剪面和物体穿过门一半时的裁切。

## 《半衰期 2》重力枪：环境里的东西都可能成为武器

### 交互细节

玩家瞄准物体后，可以把它吸到面前、保持悬浮，再把它高速发射出去。

箱子、锯片、油桶、暖气片不再只是场景装饰。锯片适合切割，油桶会爆炸，箱子可以挡住攻击，轻小物体能被快速投掷。

玩家进入房间后看到的不只是敌人和弹药，还会观察周围有什么能拿来用。环境本身变成了武器选择界面。

### 实现思路

从摄像机中心发射射线，命中带 Rigidbody 的可抓物体后，把它设为当前目标。

抓取期间可以有两种做法：

- 简单原型：把 Rigidbody 设为 Kinematic，每帧移动到摄像机前方的手持点；
- 更自然的版本：保持 Rigidbody 动态，用弹簧力或 PD Controller 把它拉向手持点。

第二种会保留重量感和碰撞反应，但需要限制最大力，避免物体剧烈抖动。

发射时解除手持状态，并沿摄像机前方调用：

```csharp
body.AddForce(cameraTransform.forward * launchForce, ForceMode.Impulse);
```

不同质量的物体可以有不同吸取速度和发射力度。还要避免玩家抓住自己、关卡地面或质量过大的物体。

## 《塞尔达传说：旷野之息》：让规则自己组合

### 交互细节

游戏中的火、风、金属、雷电、冰、水和炸药不是只为某一个谜题服务。

火能点燃草木；燃烧产生上升气流；金属能够导电；雷雨中金属装备会带来危险；冰可以在水面生成平台；炸药既能攻击也能推动物体。

玩家经常不是寻找“正确按钮”，而是先判断现场有哪些材料，再利用已经知道的规则组合解决问题。

### 实现思路

不要为每一种组合单独写脚本，例如：

```text
如果火碰到木箱，就执行木箱专用燃烧代码
```

更适合的做法是让对象拥有属性：

```text
Flammable：可燃
Conductive：导电
Freezable：可冻结
Explosive：可爆炸
Wet：潮湿
```

火焰只负责向周围发送“热量”或“点燃”效果。拥有 `Flammable` 的对象收到足够热量后进入燃烧状态；爆炸物燃烧到阈值后爆炸；附近草地再被点燃。

系统实现的关键不是组件数量，而是统一事件和状态

## 《Baba Is You》：玩家直接改写游戏规则

### 交互细节

关卡中的文字不是说明，而是可以推动的方块。

当文字排成：

```text
BABA IS YOU
WALL IS STOP
FLAG IS WIN
```

规则就会立即生效。把 `STOP` 从墙的句子里推走，墙就不再阻挡；把 `ROCK` 推到 `IS YOU` 前面，玩家控制的对象就会变成石头。

有趣的地方在于，玩家解决的不是规则之下的谜题，而是直接修改规则本身。

### 实现思路

场景可以建立在网格上，每个格子保存当前对象和文字块。

玩家每移动一步以后：

1. 扫描横向和纵向排列的文字；
2. 找出“名词 + IS + 属性”或“名词 + 动词 + 名词”结构；
3. 生成本回合的规则集合；
4. 清除上一回合的动态规则；
5. 把新属性应用到对应对象；
6. 再判断移动、胜利、失败和转换结果。

例如解析出 `WALL IS PUSH` 后，所有 Wall 对象都获得可推动属性；解析出 `ROCK IS FLAG` 后，石头对象被转换成旗帜对象。

这类系统最难的是规则冲突。比如同一个对象同时是 `PUSH` 和 `STOP`，必须提前决定处理顺序，并保证每次结果一致。

## 《Papers, Please》：把办公桌做成游戏世界

### 交互细节

玩家每天坐在边境检查窗口前，接过护照、通行证和证明文件，在桌面上拖动、翻看、对比，再决定盖“允许”还是“拒绝”。

操作本身很普通，但每个文件都占据桌面空间，玩家要在有限时间里整理信息。规则不断增加，人物又会讲述自己的处境。玩家最后按下的印章既是一次机械操作，也是一个可能影响他人命运的决定。

### 实现思路

每份文件都可以是一组数据：

```text
姓名
国籍
出生日期
证件号码
有效期
入境目的
停留时间
```

当天的规则由 `RuleSet` 保存，例如允许的国家、必须提交的文件、日期是否过期。玩家选择两个字段时，系统比较它们是否一致，并显示调查结果。

盖章只是提交最终判断。提交后，系统用规则验证玩家的决定，计算工资、罚款和剧情结果，再叫下一位角色进入。

桌面拖动可以直接使用 UGUI 的 PointerDown、Drag 和 PointerUp 事件；文件层级在拿起时移到最上方，放下时保留位置。重点不是做复杂物理，而是让文件摆放、翻找和盖章有明确手感。

## 《死亡搁浅》：把“走路”变成需要操作的事情

### 交互细节

多数游戏会自动处理角色走过碎石、斜坡和浅水。《死亡搁浅》把这些细节放大：货物重量、堆放位置、地面坡度和角色重心都会影响平衡。

玩家不是只按住前进，还要观察地面、规划路线，并在身体偏向一侧时主动拉回平衡。原本被忽略的“从这里走到那里”，变成了完整玩法。

### 实现思路

角色脚下可以向前方和左右发射多条地面检测射线，获得：

- 地面高度差；
- 坡度；
- 表面类型；
- 是否有可跨越障碍；
- 左右支撑是否平衡。

每件货物记录质量和相对角色的位置。把所有货物的质量中心加权计算，可以得到整体重心偏向。坡度、移动速度、转弯强度和重心偏移共同累积“失衡值”。

玩家按左右平衡键时，给失衡值施加反方向修正。超过阈值后播放踉跄，继续恶化才真正摔倒。这样玩家有时间发现问题并补救，而不是随机摔倒。

## 交互：把一种材质“贴”到另一件物体上

### 怎么玩

玩家有一个能够吸取表面性质的工具。对冰面使用后获得“滑”，再把它贴到墙上，墙就会变得非常滑；从弹簧垫吸取“弹性”，贴到地板上，地板就能把玩家弹起。

## 交互：物体会记住一次力量

### 怎么玩

一种特殊物体能够记住最近一次受到的冲击。玩家可以先让巨锤砸中它，它会保存这次力量；再把它搬到别处释放，重现同样方向和大小的冲击。

## 交互：只有不看它时，它才会行动

### 怎么玩

某些物体只会在摄像机看不到它时移动。玩家盯着它，它就完全静止；一转身，它可能靠近、打开门、搬动物品或改变房间结构。
