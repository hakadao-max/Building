# P01_封面

这一单元介绍普通静态模型从 Blender 到 Unity 的完整资产管线。重点不是记住所有按钮，而是建立可重复、可协作、可安全重导入的交付方式。

---

# P02_学习路线

流程分为准备、导出、导入、材质、Prefab 和验收六段。任何异常都应该沿这条链向上追溯，先修源头，再重新导出和导入。

---

# P03_为什么交付FBX

`.blend` 是制作源文件，FBX 才是明确的交换格式。直接让 Unity 读取 `.blend` 会依赖本机 Blender 版本，也无法保证程序化材质节点完整还原，因此团队交付应使用 FBX 和独立贴图。

---

# P04_静态模型导出预设

静态模型的基础预设由导出范围、变换、几何和禁用项四部分组成。先用统一值建立稳定起点，再根据项目管线对三角化、顶点色等特殊数据做有限调整。

---

# P05_路径分批与包括

路径模式推荐自动，分批模式对普通模型保持关闭。导出范围应限制到当前选中对象，类型以网格为主，只有需要根节点或挂点时才加入空物体。

---

# P06_单位轴向与变换

尺寸和朝向必须在 Blender 中正确，Unity 实例才能保持 Scale 为 1。Metric、Unit Scale 1.0、应用 Rotation & Scale、-Z 向前和 Y 向上共同构成一致的空间约定，Origin 则决定模型在游戏中的旋转支点。

---

# P07_几何数据与动画开关

普通静态模型通常导入自定义法线并应用修改器，切线由 Unity 以 MikkTSpace 计算。没有实际用途的数据不要导出，骨架和动画尤其要明确关闭，避免产生无关 Clip 与运行时负担。

---

# P08_保存预设与导出动作

把设置保存为 `Unity_静态模型` 可以减少重复操作，但预设不会替你选择当前对象。每次仍要检查对象范围、物体类型、变换、几何以及动画开关，再导出并到 Unity 验证。

---

# P09_UnityModel页签

Unity 的 Model 页签负责尺寸、轴向、法线、切线、UV 与部分运行时成本设置。Scale Factor 应保持 1，Read/Write 默认关闭，自动碰撞体只适合原型，正式资源应使用外层 Prefab 的简单碰撞方案。

---

# P10_Rig与Animation

静态模型在 Rig 页签使用 None，并关闭 Import Animation。如果 Unity 仍然出现动画 Clip，不要只在 Unity 中隐藏它，应回到 Blender 关闭导出动画并重新生成 FBX。

---

# P11_材质与贴图管线

Blender 材质节点是制作环境，Unity 材质才是最终运行时配置。Base Color 使用 sRGB，Normal 设置为 Normal Map，金属度与 AO 等数据贴图关闭 sRGB，Roughness 需要反相后才能作为 Smoothness 使用。

---

# P12_外层Prefab与职责边界

FBX 是可重新导入的模型资源，不适合直接承载游戏逻辑。外层 ObjectRoot Prefab 保存 Collider、Rigidbody 与脚本，内部 Model 节点只引用 Imported FBX，这样 Blender 更新模型时不会破坏 Unity 游戏配置。

---

# P13_快速检查与交付

最终验收应覆盖 Blender 源头、FBX 导出、Unity 导入、材质与 Prefab，以及实际场景测试。只有 Scale 为 1、朝向正确、材质正常、碰撞可用并且可以安全重导入，资产才真正达到交付标准。
