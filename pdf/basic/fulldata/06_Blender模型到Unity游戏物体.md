# Blender 导出 FBX 与 Unity 导入注意事项

本文只介绍普通静态模型从 Blender 导出 FBX 时的界面配置，以及导入 Unity 后需要检查的内容。默认不包含骨骼和动画流程。

建议向 Unity 提交明确导出的 `.fbx` 和独立贴图，不要直接把 `.blend` 文件放进 Unity。直接导入 `.blend` 会依赖本机安装的 Blender 版本，不利于团队协作和稳定构建。

---

## 一、Blender FBX 导出界面如何配置

选中需要导出的模型，然后使用：

```text
文件 > 导出 > FBX (.fbx)
```

普通静态模型可以先照下面这套配置：

```text
路径模式：自动
分批模式：关闭

限制到：勾选“选定的物体”
物体类型：只选“网格”；需要根节点或挂点时再加“空物体”
自定义属性：关闭

缩放：1.00
应用缩放：全部局部
向前：-Z 向前
向上：Y 向上
应用单位：开启
使用空间变换：开启
应用变换：关闭

动画：关闭
```

### 1. 路径模式

路径模式决定 FBX 如何记录贴图等外部文件的路径，不会把 Blender 的复杂节点材质完整转换成 Unity 材质。

| 路径模式 | 含义 | 使用建议 |
| --- | --- | --- |
| 自动 | 导出目录附近的文件使用相对路径，其他位置使用绝对路径 | 推荐。贴图在 Unity 中独立管理时保持此项即可 |
| 绝对 | 写入当前电脑上的完整路径 | 不推荐。换电脑或移动项目后容易失效 |
| 相对 | 尽量写入相对于 FBX 的路径 | FBX 和贴图始终保持固定目录关系时可用 |
| 匹配 | 按 Blender 源文件中原有的相对或绝对路径写入 | 容易继承混乱的旧路径，团队项目一般不使用 |
| 去除路径 | 只记录文件名，不记录文件夹 | 只有目标程序能按文件名找到贴图时使用；重名贴图容易匹配错误 |
| 复制 | 把引用的贴图复制到导出位置 | 需要把 FBX 和贴图一起打包交付时使用 |

推荐选择：

- 贴图已经在 Unity 项目中单独整理：选择 `自动`；
- 需要导出一个包含 FBX 和贴图的交付文件夹：选择 `复制`；
- 选择 `复制` 后，右侧会启用“嵌入贴图”按钮。Unity 项目通常不要嵌入，保持贴图为可单独管理的文件更清楚。

### 2. 分批模式

普通模型选择 `关闭`，表示把当前选择内容导出为一个 FBX。

其他分批模式会按场景或集合自动生成多个 FBX，只有批量导出大量规范化资源时才需要。右侧的文件夹按钮用于让每个批量导出的 FBX 拥有自己的文件夹；分批模式关闭时无需设置。

### 3. 包括

#### 限制到

普通导出推荐：

| 选项 | 是否开启 | 说明 |
| --- | --- | --- |
| 选定的物体 | 开启 | 只导出当前选中的模型，最直观也最不容易带入无关内容 |
| 可见物体 | 关闭 | 否则场景中所有可见物体都可能被导出 |
| 活动集合 | 关闭 | 只有按 Collection 管理整组资源时才开启 |

导出前先在 3D 视图中选好对象，再勾选 `选定的物体`。如果一个道具包含多个零件，应把需要的零件全部选中后一起导出。

#### 物体类型

截图中所有类型都处于选中状态，普通静态模型不需要这样设置。

| 物体类型 | 普通静态模型如何选择 |
| --- | --- |
| 网格 | 开启，这是实际模型 |
| 空物体 | 默认关闭；需要保留根节点、定位点或挂点时开启 |
| 摄像机 | 关闭 |
| 灯光 | 关闭 |
| 骨架 | 关闭 |
| 其他 | 默认关闭；曲线、文字等应先确认转换结果后再导出 |

`自定义属性` 默认关闭。只有 Unity 端确实编写了导入工具读取 Blender 自定义属性时才开启。

### 4. 变换

| 选项 | 推荐值 | 说明 |
| --- | --- | --- |
| 缩放 | `1.00` | 不要在导出时用 `0.01` 或 `100` 补救错误尺寸 |
| 应用缩放 | `全部局部` | 配合 `Unit Scale = 1.0` 使用，保持 FBX 全局缩放为 1 |
| 向前 | `-Z 向前` | Blender 导出到 Unity 的常用方向设置 |
| 向上 | `Y 向上` | Unity 使用 Y 轴向上 |
| 应用单位 | 开启 | 按 Blender 当前的场景单位导出 |
| 使用空间变换 | 开启 | 将 Blender 与 Unity 的轴向差异应用到对象旋转 |
| 应用变换 | 关闭 | 这是带警告的实验性选项，普通导出不要开启 |

导出前还应确认：

- Scene Units 使用 `Metric`，建议 `Unit Scale = 1.0`；
- 模型按 `1 Blender Unit = 1 米` 制作；
- 对象已经执行 `Ctrl + A > Rotation & Scale`；
- 应用后对象的 Scale 应为 `(1, 1, 1)`；
- Origin 位于合理位置，例如门的 Origin 在铰链处。

如果模型导入 Unity 后必须把 Scale 改成 `0.01` 才正常，应回 Blender 检查单位和实际尺寸，不要长期依赖 Unity 的 Transform 缩放补偿。

> 如果项目已经使用并验证了另一套“应用缩放”预设，应保持项目统一，不要让同类模型混用不同设置。

### 5. 几何数据

展开 `几何数据` 后，普通 Unity 模型推荐：

| 选项 | 推荐值 | 说明 |
| --- | --- | --- |
| 平滑 | `仅法线` | Unity 可以导入 Blender 的自定义法线 |
| 应用修改器 | 开启 | 导出修改器计算后的最终 Mesh |
| 导出细分曲面 | 关闭 | 通常直接导出已计算的最终模型 |
| 松散边 | 关闭 | Unity 游戏 Mesh 一般不需要只有边、没有面的几何 |
| 三角化面 | 默认关闭 | 重要法线烘焙资产可按项目管线统一三角化 |
| 切线空间 | 默认关闭 | 通常让 Unity 使用 MikkTSpace 计算切线 |
| 顶点颜色 | 未使用时选 `无` | Shader 使用顶点色时再按项目色彩空间导出 |

开启 `应用修改器` 会导出修改器计算后的结果。若模型使用形态键，需要先确认修改器与形态键的兼容性；普通静态模型一般没有这个问题。

### 6. 骨架与动画

本文默认导出普通静态模型：

- `物体类型` 中不要选择 `骨架`；
- `骨架` 面板无需配置；
- 取消勾选最下方的 `动画`。

截图中的 `动画` 当前为开启状态，应将它关闭，避免把无关的关键帧、Action 或场景动画写进 FBX。

### 7. 保存导出预设

界面顶部的 `操作项预设` 可以保存当前配置。建议把上述设置保存为例如：

```text
Unity_静态模型
```

以后导出同类模型时先选择该预设，再检查“选定的物体”和“物体类型”，可以减少漏改选项。

## 二、导入 Unity 后的注意事项

选中 Project 窗口中的 FBX，在 Inspector 中检查导入设置，修改后点击 `Apply`。

### 1. Model 页签

- `Scale Factor` 保持 `1`，并开启 `Convert Units`。模型实例 Scale 为 `(1, 1, 1)` 时，尺寸应正确。
- 检查模型是否横躺、反向或带有异常根节点旋转；同类资源应使用统一的 `Bake Axis Conversion` 设置。
- `Normals` 选择 `Import`，保留 Blender 中处理好的平滑和硬边。
- `Tangents` 通常选择 `Calculate Mikktspace`。使用法线贴图时不要设置为 `None`。
- `Read/Write` 默认关闭，只有运行时脚本需要读取或修改 Mesh 时才开启。
- `Mesh Compression` 先保持 `Off` 或低压缩，确认外观正常后再优化。
- `Generate Colliders` 只适合快速原型。正式资源通常在外层 Prefab 中添加简单 Collider 或专用低模碰撞体。
- 静态烘焙模型没有第二套 UV 时，可以开启 `Generate Lightmap UVs`，但仍需检查烘焙接缝和重叠。

### 2. Rig 与 Animation 页签

普通静态模型设置为：

```text
Rig > Animation Type：None
Animation > Import Animation：关闭
```

如果 Unity 仍然显示动画 Clip，说明 Blender 导出时没有关闭 `动画`，修改导出设置后重新导出即可。

### 3. Materials 与贴图

- 不要期待 Blender 程序化材质自动还原。建议在 Unity 中创建当前渲染管线对应的材质，再通过 `Remapped Materials` 映射给 FBX。
- Base Color 贴图保持 `sRGB`。
- Normal 贴图的 `Texture Type` 设置为 `Normal Map`。
- Metallic、AO、Mask 等数据贴图通常关闭 `sRGB`。
- Roughness 不能直接当作 Smoothness 使用，通常需要反相：`Smoothness = 1 - Roughness`。
- 模型变粉通常表示 Shader 与当前渲染管线不兼容；模型变白通常表示材质或贴图没有正确映射。

### 4. 不要直接修改 FBX 内部对象

FBX 是导入资源，重新导入后，直接加在其内部层级上的组件或修改可能丢失。建议创建外层 Prefab：

```text
ObjectRoot
├── Collider / Rigidbody / Gameplay Scripts
└── Model
    └── Imported FBX
```

Blender 负责 Mesh、层级、轴心和 UV；Unity 负责材质映射、Collider、Rigidbody、脚本和 Prefab。

## 三、快速检查

### Blender

- [ ] 路径模式为 `自动`，分批模式为 `关闭`；
- [ ] 开启 `选定的物体`，物体类型只选择 `网格`，需要时添加 `空物体`；
- [ ] 缩放为 `1.00`，方向为 `-Z 向前 / Y 向上`；
- [ ] 开启 `应用单位` 和 `使用空间变换`，关闭 `应用变换`；
- [ ] 几何数据按项目预设导出；
- [ ] 关闭 `骨架` 和 `动画`。

### Unity

- [ ] 实例 Scale 为 `1` 时尺寸正确，朝向和轴心正常；
- [ ] Normals、Tangents、Lightmap UV 和 Read/Write 设置合理；
- [ ] Rig 为 `None`，没有导入无关动画；
- [ ] 材质、Shader、Normal Map 与数据贴图设置正确；
- [ ] 游戏组件放在外层 Prefab 中；
- [ ] 已在实际场景中检查渲染和碰撞结果。
